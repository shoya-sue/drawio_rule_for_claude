# drawio図解生成 プロンプトテンプレート

Claudeに図解生成を依頼する際のテンプレート集です。

---

## 基本テンプレート

最もシンプルな依頼形式です。

```
## 依頼

以下の情報を元に、{図解の種類}をdrawio形式で生成してください。

## 図解の種類

{業務フロー図 / システム構成図 / 画面遷移図 / データフロー図 / ER図 / シーケンス図}

## 入力情報

{ここに元ソースを貼り付け、またはファイルパスを指定}

## 出力要件

- ファイル名: {output.drawio}
- 要素数目安: {10-15個程度}
```

---

## 図解種類別テンプレート

### 業務フロー図

```
## 依頼

以下の業務内容から業務フロー図をdrawio形式で生成してください。

## 業務内容

{業務の説明、手順、関係者などを記述}

## 含めてほしい要素

- 開始・終了ポイント
- 各処理ステップ
- 分岐条件（あれば）
- 担当者/システム（あれば）

## 出力要件

- ファイル名: business-flow.drawio
- フローの方向: 上から下 / 左から右
```

### システム構成図

```
## 依頼

以下の情報からシステム構成図をdrawio形式で生成してください。

## システム情報

{システムの説明、コンポーネント、技術スタックなどを記述}

## 含めてほしい要素

- 主要コンポーネント
- データの流れ
- 外部連携（あれば）
- ユーザー/クライアント

## 出力要件

- ファイル名: system-architecture.drawio
- レイヤー構成: あり / なし
```

### 画面遷移図

```
## 依頼

以下の情報から画面遷移図をdrawio形式で生成してください。

## 画面情報

{画面一覧、遷移条件、ユーザーアクションなどを記述}

## 含めてほしい要素

- 各画面（名前と簡易説明）
- 遷移の矢印（アクション名を含む）
- 開始画面の明示

## 出力要件

- ファイル名: screen-flow.drawio
```

### データフロー図（DFD）

```
## 依頼

以下の情報からデータフロー図をdrawio形式で生成してください。

## データ処理情報

{データの入力元、処理内容、出力先などを記述}

## 含めてほしい要素

- 外部エンティティ（四角）
- プロセス（丸）
- データストア（二重線）
- データフロー（矢印）

## 出力要件

- ファイル名: data-flow.drawio
- DFDレベル: 0 / 1 / 2
```

### ER図

```
## 依頼

以下の情報からER図をdrawio形式で生成してください。

## データ構造情報

{エンティティ、属性、リレーションなどを記述}

## 含めてほしい要素

- エンティティ（テーブル名）
- 主要属性（PK、主要カラム）
- リレーション（1:N、N:Mなど）

## 出力要件

- ファイル名: er-diagram.drawio
- 表記法: IE記法 / IDEF1X
```

### シーケンス図

```
## 依頼

以下の情報からシーケンス図をdrawio形式で生成してください。

## 処理シーケンス情報

{アクター、システム、メッセージのやり取りなどを記述}

## 含めてほしい要素

- アクター/オブジェクト（上部）

## 出力要件

- ファイル名: sequence-diagram.drawio
```

### ユースケース図

```
## 依頼

以下の情報からユースケース図をdrawio形式で生成してください。

## システム・機能情報

{システム名、アクター（利用者/外部システム）、機能一覧などを記述}

## 含めてほしい要素

- システム境界（四角枠）
- アクター（人型アイコン）
- ユースケース（楕円）
- 関連線（アクターとユースケースを接続）

## 出力要件

- ファイル名: usecase-diagram.drawio
```

### コンテキスト図

```
## 依頼

以下の情報からコンテキスト図をdrawio形式で生成してください。

## システム情報

{対象システム名、外部エンティティ（人、組織、外部システム）、データのやり取りなどを記述}

## 含めてほしい要素

- 中央にシステム（円または四角）
- 外部エンティティ（四角）
- データフロー（矢印とラベル）

## 出力要件

- ファイル名: context-diagram.drawio
```

### 状態遷移図

```
## 依頼

以下の情報から状態遷移図をdrawio形式で生成してください。

## 状態情報

{オブジェクト/エンティティ名、状態一覧、遷移条件/イベントなどを記述}

## 含めてほしい要素

- 状態（角丸四角）
- 開始状態（黒丸）

## 入力ソース別の指定方法

### テキスト/ドキュメントの場合

```
## 入力情報

以下のドキュメントを元にしてください：

"""
{ドキュメント内容をここに貼り付け}
"""
```

### コードの場合

```
## 入力情報

以下のコードの構造を図解してください：

ファイル: {ファイルパス}

または

```{言語}
{コード内容}
```
```

### 画像の場合

```
## 入力情報

添付の画像を元に図解を作成してください。

{画像ファイルを添付、またはパスを指定}

画像の説明: {何の画像か簡単に説明}
```

### 既存ファイル参照の場合

```
## 入力情報

以下のファイルを参照してください：

- {ファイルパス1}
- {ファイルパス2}
```

---

## 詳細指定オプション

必要に応じて以下を追加してください：

### 色指定

```
## スタイル指定

- メインカラー: 青系 / 緑系 / オレンジ系
- 強調したい要素: {要素名}
```

### レイアウト指定

```
## レイアウト指定

- 配置方向: 横並び / 縦並び / 階層構造
- グルーピング: あり（{グループ名}でまとめる）
```

### 詳細度指定

```
## 詳細度

- シンプル（主要要素のみ、5-7個）
- 標準（10-15個）
- 詳細（15-20個）
```

---

## クイックプロンプト

急いでいる場合の最短テンプレート：

```
{入力情報}を元に{図解の種類}を作成して。drawio形式で出力。
```

例：
```
このREADMEを元にシステム構成図を作成して。drawio形式で出力。
```

## Claudeへの補足指示（必要に応じて）

以下の指示を追加すると精度が向上します：

### 要素の明示

```
以下の要素は必ず含めてください：
- {要素1}
- {要素2}
- {要素3}
```

### 除外指定

```
以下は図に含めないでください：
- {除外要素1}
- {除外要素2}
```

### 参考イメージ

```
参考イメージ：
- {類似の図解のURL、または説明}
- 「〇〇のような形式で」
```

## 出力後の指示テンプレート

生成後に修正を依頼する場合：

### 要素追加

```
生成した図に以下を追加してください：
- {追加要素}
```

### 配置変更

```
{要素A}と{要素B}の位置を入れ替えてください。
```

### スタイル変更

```
{要素名}の色を{色}に変更してください。
```

### 矢印追加

```
{要素A}から{要素B}への矢印を追加してください。ラベルは「{ラベル名}」。
```
